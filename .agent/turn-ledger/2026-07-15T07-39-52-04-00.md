## Main finding

```txt
visual.resize()
  -> resolution.resize(cssWidth, cssHeight)
     -> effectivePixelRatio = cappedDpr * dynamicScale
     -> renderer.setPixelRatio(effectivePixelRatio)
     -> composer.setPixelRatio(effectivePixelRatio)
     -> composer.setSize(cssWidth, cssHeight)
        -> color targets use css size * effectivePixelRatio
  -> hdrComposer.resize(cssWidth, cssHeight)
     -> composer.setSize(cssWidth, cssHeight)
        -> color targets remain css size * effectivePixelRatio
     -> depthTexture.image.width = cssWidth
     -> depthTexture.image.height = cssHeight
```

The two local depth attachments are therefore explicitly rewritten to CSS dimensions while the EffectComposer render targets remain physical effective-pixel dimensions. This is a source-backed invariant gap, not a reproduced framebuffer failure.
