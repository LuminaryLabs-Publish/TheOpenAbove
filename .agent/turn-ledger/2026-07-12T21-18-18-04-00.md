## Finding

```txt
world.setResource(BalloonSnapshot, snapshot)
world.setResource(VisualSnapshot, snapshot.visual)

engine.openAbove.getState()
  -> world.getResource(BalloonSnapshot)

engine.openAbove.getVisualState()
  -> world.getResource(VisualSnapshot)

window.GameHost.getState().nexusEngine
  -> same stored BalloonSnapshot reference
```

Nexus Engine stores and returns those objects directly. The complete snapshot’s `visual` field and `VisualSnapshot` therefore alias the same writable object, and resource journal rows retain mutable `previous` and `value` references.
